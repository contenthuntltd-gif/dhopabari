# অ্যাডমিন প্যানেল + রাইডার — সেটআপ

Admin/rider এখন সব কাস্টমার দেখতে পারে, নতুন ইউজার তৈরি করতে পারে, আর যেকোনো
কাস্টমারের প্রোফাইল থেকে তার হয়ে অর্ডার দিতে পারে। কাস্টমাররাও আগের মতো নিজে
অর্ডার করতে পারে। সব অর্ডার একই `orders` টেবিলে যায়, তাই ইতিহাস এক জায়গাতেই থাকে।

কোড লেখা শেষ। নিচের ৪টা ধাপ **আপনাকে** করতে হবে — এগুলো কোড থেকে করা যায় না,
কারণ এতে Supabase-এর `service_role` কী লাগে।

---

## ১. মাইগ্রেশন চালান

Supabase Dashboard → **SQL Editor** → নিচের ফাইলগুলো **এই ক্রমে** চালান:

| ফাইল | কী করে |
|---|---|
| `migrations/0001_auth_profiles.sql` | (আগে চালানো থাকলে বাদ দিন) `profiles` টেবিল |
| `migrations/0002_orders_and_staff.sql` | `orders` টেবিল, রোল হেল্পার, স্টাফ RLS পলিসি |

`0002` চালালে যা তৈরি হয়:
- `public.orders` — অর্ডার ইতিহাসের আসল উৎস
- `is_staff()` / `is_admin()` — RLS পলিসিতে রোল চেক করার হেল্পার
- স্টাফ পলিসি — admin/rider সব প্রোফাইল ও সব অর্ডার দেখতে পাবে
- `customer_summary` ভিউ — কাস্টমার তালিকায় অর্ডার সংখ্যা ও মোট খরচ

## ২. প্রথম অ্যাডমিন বানান

মুরগি-ডিমের সমস্যা: শুধু অ্যাডমিনই স্টাফ বানাতে পারে, তাই প্রথমজনকে হাতে বানাতে হয়।

1. অ্যাপে ঢুকে **সাধারণভাবে রেজিস্টার করুন** (ফোন + পাসওয়ার্ড) — যে নম্বরটা
   অ্যাডমিন হবে সেটা দিয়ে।
2. `migrations/0003_bootstrap_first_admin.sql` খুলে `target_phone` এ আপনার
   আসল নম্বর বসান (`+8801712345678` ফরম্যাটে)।
3. SQL Editor-এ ফাইলটা চালান।

> ⚠️ `auth.users` টেবিলে সরাসরি INSERT করবেন না — Supabase পাসওয়ার্ড হ্যাশ করে
> রাখে, হাতে বসানো row দিয়ে লগইন হবে না।

এরপর বাকি সব অ্যাডমিন ও রাইডার প্যানেল থেকেই তৈরি হবে, আর SQL ছোঁয়া লাগবে না।

## ৩. Edge Function ডিপ্লয় করুন

অন্য কারো অ্যাকাউন্ট পাসওয়ার্ড দিয়ে তৈরি করতে `service_role` কী লাগে। **ওই কী
কখনোই Flutter অ্যাপে রাখা যাবে না** — web bundle / APK থেকে যে কেউ বের করে পুরো
ডাটাবেসের full access পেয়ে যাবে। তাই কাজটা সার্ভারে হয়:

```bash
npx supabase login
npx supabase link --project-ref stxzqmrnezedphysmczq
npx supabase functions deploy admin-create-user
npx supabase functions deploy admin-manage-user
```

| ফাংশন | কাজ |
|---|---|
| `admin-create-user` | নতুন কাস্টমার/রাইডার অ্যাকাউন্ট (ফোন + পাসওয়ার্ড) |
| `admin-manage-user` | পাসওয়ার্ড পরিবর্তন, অ্যাকাউন্ট ডিলিট |

`SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` — এই তিনটা
Supabase নিজে থেকেই Edge Function-এ দিয়ে দেয়, আলাদা করে সেট করতে হবে না।

## ৪. পরীক্ষা করুন

1. অ্যাডমিন নম্বর + পাসওয়ার্ড দিয়ে লগইন করুন (লগইন স্ক্রিন → ⋮ → অ্যাডমিন)
2. কাস্টমার তালিকা → **নতুন কাস্টমার** → নাম, নম্বর, পাসওয়ার্ড, ঠিকানা দিন
3. ওই কাস্টমারে ক্লিক → **এই কাস্টমারের অর্ডার** → অর্ডার তৈরি করুন
4. লগআউট করে ওই কাস্টমারের নম্বর + পাসওয়ার্ড দিয়ে লগইন করুন — অর্ডারটা তার
   ইতিহাসে দেখা যাবে

## নিরাপত্তা — কে কী পারে

RLS ডাটাবেসে চালু, তাই এই নিয়মগুলো ক্লায়েন্ট নয়, সার্ভার মানে:

| | কাস্টমার | রাইডার | অ্যাডমিন |
|---|---|---|---|
| নিজের প্রোফাইল দেখা/এডিট | ✅ | ✅ | ✅ |
| সব কাস্টমার দেখা | ❌ | ✅ | ✅ |
| নতুন কাস্টমার তৈরি | ❌ | ✅ | ✅ |
| নতুন রাইডার/অ্যাডমিন তৈরি | ❌ | ❌ | ✅ |
| নিজের অর্ডার দেখা | ✅ | ✅ | ✅ |
| সব অর্ডার দেখা | ❌ | ✅ | ✅ |
| অন্যের হয়ে অর্ডার | ❌ | ✅ | ✅ |
| অর্ডার স্ট্যাটাস বদলানো | ❌ | ✅ | ✅ |
| অর্ডার ডিলিট | ❌ | ❌ | ✅ |

**আপনার সিদ্ধান্ত অনুযায়ী রাইডারের ক্ষমতা অ্যাডমিনের সমান রাখা হয়েছে** — মানে
প্রতিটি রাইডার প্রত্যেক কাস্টমারের ফোন নম্বর আর বাসার ঠিকানা দেখতে পাবে। পরে
সীমিত করতে চাইলে `0002` মাইগ্রেশনের `profiles_select_staff` পলিসিটা বদলাতে হবে।

একটা ট্রিগার (`guard_role_change`) আলাদাভাবে আটকায় যাতে কোনো কাস্টমার নিজের
রোল বদলে নিজেকে অ্যাডমিন বানাতে না পারে।

## যা এখনো তৈরি হয়নি

এই কাজের অংশ ছিল না, এখনো mock ডেটায় চলছে:

- **রাইডার ওয়ালেট, আয়, উইথড্রয়াল** — কোনো payouts টেবিল নেই। ওয়ালেটে `৳0`-র
  বদলে `—` দেখানো হয়, যাতে ভুল বোঝা না যায়।
- **রাইডার রেটিং** — কোথাও রেটিং জমা হয় না।
- **রাইডার অনলাইন/অফলাইন** — presence টেবিল নেই, টগলটা শুধু ওই সেশনে থাকে।
- **ক্যাটালগ (আইটেম ও দাম)** — এখনো `mock_data.dart`-এ। অর্ডারে দাম snapshot
  হিসেবে জমা হয়, তাই পরে দাম বদলালেও পুরনো অর্ডার ঠিক থাকবে।
